# 项目启动书：校园实验室预约管理系统

## 项目概况

项目名称：校园实验室预约管理系统

开发模式：前后端分离

技术栈：后端 JAVA，前端 Vue.js

主要受众：校学生（用户）及实验室管理员

## 背景与意义

### 现状与问题

预约混乱：实验室使用依赖线下登记、口头报备，易出现时段冲突、重复占用，资源分配无序。

管理低效：管理员人工统计预约信息、核对使用情况，耗时费力，数据易出错、追溯困难。

使用不便：学生无法实时查看实验室空闲状态，预约流程繁琐，取消/改期操作复杂。

资源浪费：实验室开放时段、设备状态无统一公示，闲置资源无法被高效利用。

### 项目意义

规范流程：实现实验室预约全流程数字化，杜绝时段冲突，保障使用秩序。

提升效率：简化学生预约操作，降低管理员人工管理成本，提升实验室资源利用率。

数据透明：实时展示实验室状态、预约记录，实现信息公开可查。

数字化校园：完善校园教学资源管理体系，助力校园智慧化建设。  

## 项目目标

实现预约数字化：支持学生按实验室、时段自主预约，自动校验冲突。

优化管理流程：为管理员提供可视化后台，统一管理实验室、时段与预约信息。

提升使用体验：学生可快速查询空闲资源、管理个人预约，操作便捷高效。

保障资源有序：规范实验室使用规则，减少资源浪费与使用纠纷。

## 系统功能范围

### 用户端（前台）

|     |     |
| --- | --- |
| 功能模块 | 功能描述 |
| 登录注册 | 支持学生学号注册、登录，是预约和查看记录的前提。 |
| 实验室查询 | 按实验室名称、类型、位置筛选，查看详情与空闲时段。 |
| 时段预约 | 选择目标实验室与可用时段，提交预约申请。 |
| 预约管理 | 查看个人预约记录，支持未使用时段的预约取消。 |
| 个人中心 | 统一管理个人预约信息、账号信息。 |
| 使用提醒 | 查看预约成功、即将到期、取消成功等状态提醒。 |

### 管理端（后台）

|     |     |
| --- | --- |
| 功能模块 | 功能描述 |
| 实验室信息管理 | 对实验室名称、位置、类型、开放状态进行增、删、改、查。 |
| 预约时段管理 | 配置实验室可预约时间段、每日可约人数、使用时长限制。 |
| 预约信息管理 | 查看所有预约记录，审核异常预约，取消违规预约。 |
| 用户信息管理 | 管理学生账号，查看用户预约行为与使用记录。 |

## 业务约束与非功能需求

业务规则：同一实验室同一时间段内不可重复预约；学生单日预约时长不超过上限。

数据维护：管理员实时更新实验室开放状态、时段规则，保证信息准确有效。

核心指标：减少预约冲突率，提升实验室日均使用率，降低管理员管理耗时。

非功能需求：系统响应快速、运行稳定，操作界面简洁易懂，适配校园网络环境。。
